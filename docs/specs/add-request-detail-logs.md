# Add Request Json Body Detail Logs

日志详情页新增请求 Json Body 的日志展示，方便用户查看请求的详细信息。

## 1. 需求背景

目前日志详情页仅展示了请求的基本信息，如请求 URL、请求方法、请求头等，但缺乏对请求 Json Body 的展示。对于一些需要查看请求内容的用户来说，这会带来不便。因此，我们计划在日志详情页新增请求 Json Body 的日志展示，以提供更全面的日志信息。

## 2. 需求分析
### 2.1 功能需求
- 在日志详情页新增一个区域用于展示请求 Json Body。
- 该区域应支持格式化显示 Json 数据，便于用户阅读。
- 新增的请求 Json Body 日志展示功能应与现有的日志详情页设计保持一致，确保用户体验的一致性。
- 新增一个选项，允许用户选择是否记录请求日志详情，并根据该选项决定是否记录请求 Json Body 的内容。

### 2.2 非功能需求
- 新增的请求 Json Body 日志展示功能应具有良好的性能，确保在日志详情页加载时不会显著增加加载时间。
- 该功能应兼容主流浏览器，并适应不同屏幕尺寸的设备。

## 3. 实现方案
### 3.1 前端实现
- 在日志详情页的前端代码中，新增一个组件用于展示请求 Json Body
- 使用成熟的第三方库（如 `react-json-view`）来格式化和展示 Json 数据。

### 3.2 后端实现
- 在后端日志记录逻辑中，新增一个字段用于存储请求 Json Body
- 确保在记录日志时正确捕获和存储请求 Json Body 的内容。
- 在日志详情接口中，新增一个字段用于返回请求 Json Body 的内容。 
- 在设置 `日志维护` 中新增一个选项，允许用户选择是否记录请求日志详情，并根据该选项决定是否记录请求 Json Body 的内容。
- 确保在日志详情接口中正确处理请求 Json Body 的内容，避免因内容过大或格式不正确导致接口响应失败。
- 在日志详情接口中，新增一个字段用于返回请求 Json Body 的内容，并确保该字段的内容在日志详情页正确展示。
